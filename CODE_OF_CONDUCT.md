# Code of Conduct

## Cel

Chcemy, aby praca nad projektem była rzeczowa, spokojna i skupiona na rozwiązaniach. Ten dokument dotyczy issue, pull requestów, dyskusji i innych kanałów związanych z PicOrgFTP-SQL.

## Oczekiwane zachowania

- Pisz konkretnie i z szacunkiem.
- Krytykuj kod, decyzję techniczną albo proces, a nie osobę.
- Zakładaj dobrą wolę, ale proś o doprecyzowanie, gdy coś jest niejasne.
- Zgłaszaj błędy z krokami odtworzenia, logami i kontekstem środowiska, jeżeli możesz je bezpiecznie udostępnić.
- Nie publikuj cudzych danych, haseł, tokenów, kluczy API ani informacji z prywatnych środowisk.

## Nieakceptowane zachowania

- Obrażanie, nękanie, groźby i ataki personalne.
- Publikowanie prywatnych danych bez zgody.
- Celowe zaśmiecanie dyskusji albo utrudnianie pracy nad projektem.
- Publiczne publikowanie podatności przed daniem maintainerowi szansy na reakcję.

## Egzekwowanie

Maintainer może edytować albo usuwać komentarze, zamykać dyskusje, blokować użytkowników albo odrzucać wkład, jeżeli narusza te zasady.

Problemy bezpieczeństwa zgłaszaj zgodnie z [SECURITY.md](SECURITY.md), a nie przez publiczną dyskusję.
